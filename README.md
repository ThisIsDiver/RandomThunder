RandomThunder ver. 0.0.2 Reserve
=====================================
EN
=====================================

Do you have a lot of vehicles, but can't decide? What if I told you that you don't have to decide? I present to you...

**RandomThunder** is the simplest software for selecting vehicles and creating setups. With this program, the computer itself will select a crazy setup, which only remains to be assembled.

Installation
=====================================
For stable operation of **RandomThunder** you need to download and install [Python 3.11.4](https://www.python.org/downloads/release/python-3114/) (and later). The software has not been tested with earlier versions.

Have you installed Python? Download [RandomThunder](https://github.com/ThisIsDiver/RandomThunder/releases/download/RandomThunder/RandomThunder.0.0.1.Reserve.zip). After downloading the archive, the software must be unzipped to a convenient location. The main thing is that all **RandomThunder** files are in one directory (folder).

**RandomThunder** works on: **Microsoft Windows 10/11**, **GNU/Linux**. It can be launched on other operating systems, but the software may be unstable.

Customization
=====================================

Unfortunately, due to the many possible input data, **RandomThunder** only contains the vehicles from the "Researchable vehicles" column, which is available in the "Research" tab, but don't be upset! You can add the missing vehicles to the table yourself. To do this, you need to:

- Go to the "factions" directory

![image](https://github.com/user-attachments/assets/97bd67ec-1b4b-4a12-bad1-4bc376e30c51)

- Select a faction

![image](https://github.com/user-attachments/assets/7b22a376-e98e-46df-b18a-0229d024e1bf)

- Open the required table

![image](https://github.com/user-attachments/assets/5a629282-af5f-43b4-a0d9-1615f9eb7188)

- Scroll to the desired BR and at the end enter in the following format:

```[Name] - [BR]```

![image](https://github.com/user-attachments/assets/a6be21ef-743c-4c39-8fca-23235024e874)

- After that, all that remains is to save.

After you make the changes, **it is recommended to check for correctness** by running ```start.py``` and writing ```Check```. If something is wrong, the correctness check will stop before the problematic line and write the path to the file that causes the error.

![image](https://github.com/user-attachments/assets/c8d16f38-df07-491f-b153-540ad48ed3f6)

It is important to understand that the separator is ' - ' and an extra character (or an unfinished BR) can break the software. Also beware of using '-' with spaces in the technique name. This can also make the software unstable.

Кастомизация
=====================================
К сожалению из-за многих возможных вводных данных **RandomThunder** лишь содержит в себе технику из столбца "Исследуемая техника", которая доступна во вкладке "Исследования", но не стоит расстраиваться! Вы можете сами добавить недостающую технику в таблицу. Для этого необходимо:

- Перейти в директорию "factions"

![image](https://github.com/user-attachments/assets/97bd67ec-1b4b-4a12-bad1-4bc376e30c51)

- Выбрать фракцию

![image](https://github.com/user-attachments/assets/7b22a376-e98e-46df-b18a-0229d024e1bf)


- Открыть необходимую таблицу

![image](https://github.com/user-attachments/assets/5a629282-af5f-43b4-a0d9-1615f9eb7188)


- Пролистать до нужного БР и в его конце написать в следующем формате:

```[Название] - [БР]```

![image](https://github.com/user-attachments/assets/a6be21ef-743c-4c39-8fca-23235024e874)

- После этого остаётся лишь сохранить.

 После введённых вами изменений **рекомендуется провести проверку на корректность**, запустив ```start.py``` и написав ```Check```. Если что-то будет не так, то проверка на корректность остановится перед проблемной строкой и напишет путь к файлу, который вызывает ошибку.

 ![image](https://github.com/user-attachments/assets/c8d16f38-df07-491f-b153-540ad48ed3f6)


 Важно понимать, что разделителем является ' - ' и лишний символ (или недописанный БР) может сломать программу. Также остерегайтесь использования '-' с пробелами в названии техники. Это тоже может сделать программу нестабильной.
